# Contributing

Thank you for your interest in contributing to the CloudOps Enterprise Platform.

## Contribution Guidelines

- Create a new feature branch before making changes.
- Write clear and meaningful commit messages following the Conventional Commits specification.
- Test all scripts before committing.
- Update documentation whenever functionality changes.
- Keep commits focused on a single logical change.
- Submit changes through a Pull Request for review.

## Branch Naming Convention

- feature/<feature-name>
- bugfix/<issue-name>
- hotfix/<issue-name>
- docs/<documentation-update>

## Commit Message Format

Examples:

feat(terraform): provision AWS VPC

fix(docker): resolve container networking issue

docs(readme): update project overview

## Code Quality

- Follow Infrastructure as Code best practices.
- Avoid committing secrets or credentials.
- Keep the repository clean and organized.

Thank you for helping improve this project.
